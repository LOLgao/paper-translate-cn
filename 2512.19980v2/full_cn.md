# 代码大语言模型的神经元导向解释：何处、为何与如何？

ZHE YIN，上海交通大学，中国

XIAODONG GU，上海交通大学，中国

BEIJUN SHEN∗，上海交通大学，中国

代码语言模型已在广泛的代码智能任务中展现出强大的能力。尽管现有研究大多关注于基准数据集上的性能提升，但很少有研究聚焦于模型的内在可解释性——即特定神经元如何影响语法和语义等语言特征，这对于模型的透明度、可控性和可靠性至关重要。尽管自然语言处理领域已开发出多种神经元可解释性技术，但由于编程语言具有形式化结构、层次化组织和可执行性等独特特性，直接将其应用于源代码往往效果不佳。在本工作中，我们通过实证研究在神经元层面探究代码大语言模型的内在机制，旨在定位语言特定神经元（即对个别编程语言选择性响应的神经元）和概念层（即编码与语言无关的代码表示的前馈层）。我们的研究采用了两个最先进的模型，Llama-3.1-8B 和 Qwen2.5-Coder-32B，并覆盖了五种编程语言：${ \mathrm { C } } { + } { + }$、Java、Python、Go 和 JavaScript。通过分析模型对多语言代码输入的神经元激活模式，我们探究了单个神经元的作用以及不同层在输出生成过程中的贡献。我们的实证研究发现：（1）代码大语言模型包含针对个别编程语言特化的神经元，同时也存在支持通用代码生成的通用神经元子集；（2）较低层主要编码语言特定的句法结构，而中间层则捕获跨语言泛化的语义抽象，表现为概念层。为了证明这些发现的实际可用性，我们将研究成果应用于三个下游任务：面向代码生成的神经元引导微调、使用概念层嵌入的克隆检测，以及面向代码摘要的概念层表示引导的迁移学习。实验评估表明，每种策略都能持续提升多语言代码大语言模型的性能。

# 1 引言

在大量代码库上训练的大型语言模型已成为软件开发中强大的生产力工具。它们在多种代码相关任务中展现出卓越性能，例如代码补全[16]、摘要生成[44]、代码翻译[53]和代码问答[25]。然而，当前针对代码大语言模型的研究主要聚焦于提升基准测试的预测准确率，通常将模型视为黑箱。例如，约 $9 6 \%$ 的研究优先关注性能提升[21]，而很大程度上忽略了对内部决策机制的分析。这种狭隘的关注点导致对代码大语言模型内部工作机制的理解严重不足，可能损害模型的可控性并降低其可信度[12]。

近年来，代码大语言模型的可解释性已引起学术界和工业界的广泛关注。尽管如此，仍存在重大挑战阻碍着对这些模型内部工作机制更深入、更可靠的理解。首先，现有解释方法的可扩展性和计算效率往往难以跟上现代大语言模型规模和复杂性的快速增长。其次，代码模型的可解释性缺乏既定的“真实基准”，使得难以客观评估解释输出的准确性和质量[60]。第三，代码大语言模型展现出受代码语法、控制流和数据依赖关系塑造的、编程语言特有的特性[26]。因此，简单地迁移自然语言处理领域的可解释性技术常常导致次优性能[30]。例如，如第2.2节所示，直接使用LAPE[46]被证明无法有效准确识别编程语言特定的神经元。这些挑战要求一种新颖的跨学科方法，将深度学习可解释性与编程语言及编码习惯的原则相结合。我们的目标是将关注点从“哪些标记重要”转向“内部表示如何映射并操作底层程序结构”。

本文旨在研究代码大语言模型的内部机制，以识别编码编程语言特定语义表示的网络区域，并利用这些洞见来提升下游代码相关任务的性能。具体而言，我们从神经元和层级两个层面审视代码大语言模型。RQ1从内部神经元激活的角度探究模型的行为。RQ2考察模型的输出如何通过其前馈层构建，并精确定位捕获代码抽象的、语言无关语义的潜在“概念层”。基于这些发现，RQ3侧重于利用这些洞见来增强下游任务的性能，例如代码生成、克隆检测和代码摘要。我们的研究采用了两个公开可用的大语言模型，即Llama-3.1-8B和Qwen2.5-Coder-32B，以及五种流行的编程语言：$\mathrm { C } { + } { + }$、Java、Python、Go和JavaScript。

我们探讨以下研究问题：

• RQ1：代码大语言模型中是否存在编程语言特定的神经元？

动机。代码大语言模型中的语言特定神经元有助于在适应特定语言时，通过选择性冻结非专门化神经元来增强参数高效的微调。

方法。在此研究问题中，我们提出了编程语言专门化评分，这是一种识别专用于单个编程语言的神经元的新颖度量标准。PLS评分通过评估神经元的激活强度以及损失相对于该激活的梯度，来识别对特定语言至关重要的神经元。随后，我们通过基于消融的干预来验证这些神经元的语言特定性。

结果。代码大语言模型包含专用于单个编程语言的神经元，以及一组对通用代码生成至关重要的独特通用神经元。值得注意的是，这些神经表示反映了编程语言之间的系统发育关系。

• RQ2：代码大语言模型中是否存在语言无关的概念层？

动机。概念层既可以作为代码嵌入的强大语义枢纽，也可以作为跨语言知识迁移的桥梁。

方法。在此研究问题中，我们采用双管齐下的分析框架，通过语义探测和句法探测来三角定位潜在的概念层。我们使用表示相似性分析作为语义探测，使用抽象语法树节点预测作为句法探测，分析这两种属性在层级上的表示不变性和保真度。

结果。代码大语言模型以分层方式组织知识，在中间层形成概念层。它们对词汇和句法变化展现出最大的表示不变性，同时在编码细粒度句法结构时表现出最低的保真度。

• RQ3：我们如何利用代码大语言模型的可解释性来增强代码相关任务？

动机。我们在多个代码相关任务中展示了关于语言特定神经元和概念层的洞见的实际效用。

方法。在此研究问题中，我们利用RQ1和RQ2的发现来增强三个下游软件工程任务：通过神经元引导的微调进行代码生成、使用概念层嵌入进行克隆检测，以及通过概念层引导的迁移学习进行代码摘要。

结果。我们的发现提供了经验证据，表明利用机制性洞见能显著提升所有三项任务的性能。具体而言，神经元引导的微调通过减轻灾难性遗忘而优于LoRA，概念层嵌入实现了有效的零样本语义克隆检测，概念层引导的迁移学习在低资源代码摘要任务上促成了更优的性能。

# 2 背景与动机

# 2.1 大语言模型可解释性

大语言模型可解释性旨在揭示语言模型的内部机制、决策过程与表征结构，使其行为对人类而言更具可理解性、可信度与可控性[60]。该领域研究沿着若干相互关联的路径发展演进。

一个核心研究方向是对模型表征的探测与分析。探测技术通过分析特定注意力头[3]、解码嵌入概念[31]或应用表征工程[56, 62]等方法，检验模型激活中编码的信息。这还包括直接采用对数透镜式方法，将激活解码为词汇标记，以理解不同层级和位置所表征的信息[9]。这些技术为理解大语言模型处理与组织信息的微妙方式提供了洞见。

进一步的研究深入到对组件的更细粒度分析，旨在阐释神经元和注意力头等基本单元的功能。研究已成功将单个神经元映射到人类可解释的概念[11, 39]。例如，Tang等人[46]提出了语言激活概率熵，这是一种通过测量神经元在不同语言间激活的熵值来识别语言特异性神经元的指标。超越单个组件，越来越多的研究聚焦于电路级分析，探究分布式神经元群与注意力头如何交互以执行特定计算。例如，研究已识别出负责间接宾语识别等任务的电路[54]。更广泛而言，此方法可用于功能定位，例如精确定位负责

# 2.2 研究动机

尽管针对通用大语言模型已提出了多种可解释性技术，但由于编程语言中代码的独特性，直接将其应用于代码大语言模型往往不切实际[30]。与自然语言不同，编程语言通常共享常见的结构，例如循环（'for'、'while'），这可能导致基于熵的度量方法将代表这些共享概念的神经元错误地归类为与语言无关的。此外，跨语言的词汇和子词符重叠（例如变量名和运算符）会使现有技术的熵计算产生偏差。

为验证这一挑战，我们使用LAPE[46]——一种针对自然语言的可解释性技术——对代码大语言模型进行了初步研究。LAPE为每种语言识别出一组数量少且可变的神经元，仅占模型总容量的很小一部分。我们将LAPE应用于Llama-3.1-8B模型，覆盖五种编程语言：$\mathrm { C } { + } { + }$、Java、Python、Go和JavaScript。如表1左侧所示，检测到的语言特定神经元总共仅占所有神经元的$0 . 7 5 \%$。为验证这些神经元，我们停用了与每种语言相关的神经元子集，并评估其对代码生成的影响。性能评估在HumanEval-X[61]基准测试上进行，采用pass@3指标。表1右侧呈现的结果表明，这些消融操作并未在目标语言中持续导致性能下降，有时甚至提升了其他语言的性能。这种不一致性表明，LAPE所强调的神经元并非可靠的语言特定神经元。支持这一观察的是，近期研究[22]同样发现，隔离仅属于单一语言的神经元尤其具有挑战性，特别是对于像Java和$\mathrm { C } { + } { + }$这样密切相关的语言。

这些因素促使我们采用将深度学习可解释性与编程语言固有属性相结合的研究方法，以解释代码大语言模型。

# 3 实验设置



我们研究代码大语言模型的内在机制，以理解单个神经元如何影响语言特征。本节将讨论实验所使用的模型和编程语言。

# 3.1 研究对象语言模型



我们的研究基于两个公开可用的LLM进行：Llama-3.1-8B [6] 和 Qwen2.5-Coder-32B [18]。Llama-3.1 是一个在广泛文本和代码语料库上进行预训练的基础模型，而 Qwen2.5-Coder 则是一个专门针对代码的模型，其训练数据完全来自超过 3 万亿的高质量代码标记。两者均为仅解码器的自回归 Transformer 模型。我们使用 Llama-3.1 的 8B 版本（${ \sim } 1 . 5 { \mathrm { M } }$ 个神经元）作为适合微调的轻量级骨干模型。对于 Qwen2.5-Coder，我们选择 32B 变体（$\mathord { \left. \sim \right.} 4 . 8 M $ 个神经元）以考察更大规模的模型。我们仅使用每个模型的基础版本，以避免来自训练后对齐技术（如 RLHF）的混杂效应。这使得我们能够对预训练过程中内在获取的知识和架构特性进行直接且无偏的分析。

# 3.2 选定的编程语言



本研究考察了五种广泛采用的编程语言，包括 $\mathrm { C } { + } { + }$ 、Java、Python、Go 和 JavaScript。这些语言因其显著的行业影响力及独特的语言特性而被选中。该集合既包含静态类型语言（ $( \mathrm { C } + +$ 、Java、Go），也包含动态类型语言（Python、JavaScript），支持面向对象、过程式和脚本编程等多种范式。同时，它涵盖了不同的执行模型，从需要手动内存管理的编译型语言（ $\left( \mathbf { C } + + \mathbf { \right) }$ ）到具有垃圾回收机制的解释型或即时编译型语言（Python、JavaScript、Go）。这种多样性为评估代码大语言模型如何理解、区分和泛化不同编程语言提供了全面的基础。

# 3.3 实现细节

所有实验均在标准化的软硬件环境中进行，以确保可复现性。我们使用了 HuggingFace transformers [57]（v4.43.0）和 PyTorch [35]（v2.3.0）库。实验平台是一台配备两块 NVIDIA GeForce RTX 4090 GPU 的工作站，运行在 CUDA 12.2 上。所有基础模型均直接从 Hugging Face Hub 加载。为了优化计算效率和内存使用，所有训练和推理操作均使用 torch.bfloat16 混合精度格式执行。

对于所有微调任务，除非另有说明，我们采用一致的训练配置。我们使用 AdamW 优化器 [27]，学习率为 $5 \times 1 0 ^ { - 5 }$，并采用带有 $1 0 \%$ 预热阶段的线性调度器。我们使用的最大序列长度为 1,024 个标记，每个设备的批次大小为 4，并通过 2 步梯度累积实现 16 的有效批次大小。所有实验均使用固定的随机种子 42，以确保结果具有确定性。每个下游任务的具体超参数，例如训练轮数和具体的微调策略，详见第 6 节。

（2）特异性验证。对每个候选神经元，令$L ^ { * }$为其PLS评分最高的语言：

$$
L ^ {*} = \arg \max  _ {L _ {i} \in \mathbb {L}} \operatorname {P L S} (n, L _ {i}). \tag {3}
$$

当该评分至少达到次高评分的1.5倍时，该神经元被判定为语言特异性神经元，此举平衡了精确度与覆盖率。

（3）排序与选择。按最大PLS评分对神经元排序，选取前$1 \%$个归类为语言专化神经元，每个神经元对应其专属的$L ^ { * }$。

该流程最终得到的神经元集合不仅对模型预测具有高响应度和影响力，同时明确表现出对特定编程语言的专化性。

4.1.2 验证语言特异性神经元。我们采用基于消融的标准干预协议[5]来验证代码生成中识别的语言特异性神经元。该方法通过扰动神经元激活并评估对生成代码质量的影响实现：

（1）干预：在代码生成任务的前向传播过程中，我们通过将先前识别的专化神经元（如Python专用神经元）激活值在计算后立即置零来进行消融。该干预作用于神经元所在的神经模块（包括自注意力层和前馈层），从而消除其对后续层及最终输出的贡献。  
（2）评估：我们在HumanEval-X[61]基准测试上评估模型性能，该多语言数据集用于评估跨语言代码合成能力。针对每个问题，模型生成三个候选解决方案，性能通过通过率$@ 3$指标量化，该指标衡量三个方案中至少有一个通过所有单元测试的概率。

我们将基于PLS评分的消融结果与随机消融同等数量神经元的结果进行对比。若目标语言性能显著下降而其他语言受影响最小，则表明语言特异性神经元的识别具有较高准确性。

# 5 RQ2：逐层功能特性分析



在识别出语言特定神经元后，我们进一步研究代码大语言模型是否展现出语言无关的“概念层”，这些层编码了代码的抽象、语言无关的表征。这些已识别的概念层可作为代码嵌入的强大语义枢纽，以及跨语言知识迁移的桥梁。我们重点考察Transformer模型的前馈层。自然语言处理领域的先前研究[20, 48, 51]表明，前馈网络在大语言模型中充当关键的记忆组件，发展出一种特征层级结构，该结构从较低层的表层语法逐渐过渡到较高层的更抽象语义。

# 5.1 研究设计

我们引入一种双管齐下的分析框架，通过互补的语义和句法探针来识别候选概念层。首先，我们应用表征相似性分析（RSA）[24]作为语义探针，以度量逐层表征对词汇变化和句法改变的不变性，这是抽象程度的一个关键指标。其次，我们部署一个基于AST节点预测的句法探针，以追踪语法结构在各层中的编码方式。预期是，一个真正的概念层应表现出高度的语义不变性，同时表现出对细粒度句法细节的敏感性降低。

5.1.1 探测语义抽象。我们应用RSA在HumanEval-X [61]基准上研究代码LLM中的语义抽象。RSA通过分析模型各层间的表征几何结构，能够识别概念层。其核心前提是，概念层应捕获语义等价性：具有相同功能的代码片段应映射到相似的表征，即使它们在表面形式上存在差异（例如，变量名或句法结构）。

为了探测这一特性，我们设计了两个针对不同类型代码修改的受控实验。通过检查某些层在这些变化下是否表现出一致的表征，我们可以识别哪些层在代码LLM中充当语义抽象的候选层。

第一个实验测试对词汇变化的语内不变性，评估模型是否不受单一编程语言内表面标识符更改的影响。忽略这种琐碎差异的能力对于真正的语义理解至关重要。如果表征仅因变量重命名而发生显著变化，则表明其未能捕获底层的算法逻辑。对于每个原始函数 $F _ { \mathrm { o r i g } }$ ，我们通过将所有用户定义的标识符替换为随机字符串，生成一个语义等效的版本 $F _ { \mathrm { a n o n } }$ 。具体来说，我们只重命名用户定义的变量（例如，局部变量、参数和字段），而保留语言关键字和库符号不变。每个标识符被替换为一个长度为2-7的短随机字母数字字符串，以避免引入分布外的命名模式。重命名后，我们执行原始的HumanEval-X单元测试，并仅保留通过所有测试用例的函数，确保 $F _ { \mathrm { o r i g } }$ 和 $F _ { \mathrm { a n o n } }$ 在句法上有效且在行为上等效。然后，我们测量每对 $( F _ { \mathrm { o r i g } } , F _ { \mathrm { a n o n } } )$ 之间的表征相似性。

第二个实验评估句法变化下的跨语言等价性，通过评估主要句法变化下的表征不变性，直接探测与语言无关的概念层的存在。通过测量用不同语言（例如Python和Java）编写的语义等效函数之间的相似性，我们衡量模型超越不同语法、关键字和类型系统的能力。此处的高相似性将强烈表明模型捕获了一个抽象的算法核心，独立于其句法表达。我们使用HumanEval-X基准的平行语料库来比较用不同语言解决同一问题的函数对（例如 $( F _ { \mathrm { p y t h o n } } , F _ { \mathrm { j a v a } } ) _ { \mathrm { . } }$ ）。

对于这两个实验，执行以下步骤以量化逐层表征相似性：

(1) 表征提取：每个代码片段由冻结模型处理。对于每个Transformer层，我们提取每个输入词元的最终隐藏状态向量。
(2) 嵌入聚合：为了在特定层获得整个代码片段的单一向量表征，对词元向量序列应用平均池化。这种标准聚合方法为层??的代码片段??生成一个固定大小的嵌入 $V _ { l } ( s )$ 。
(3) 相似性量化：我们使用余弦相似度计算两个代码片段 $s _ { A }$ 和 $s _ { B }$ 在每个层的表征之间的相似性，该相似度衡量嵌入向量之间的角度接近度。对于给定的一对，逐层相似性定义为：

$$
\operatorname {S i m} \left(V _ {l} \left(s _ {A}\right), V _ {l} \left(s _ {B}\right)\right) = \frac {V _ {l} \left(s _ {A}\right) \cdot V _ {l} \left(s _ {B}\right)}{\left| V _ {l} \left(s _ {A}\right) \right| \left| V _ {l} \left(s _ {B}\right) \right|}. \tag {4}
$$

这些相似性分数在所有代码对之间取平均值。由此产生的语内不变性分数量化了模型对表面词汇变化的鲁棒性，而跨语言等价性分数则评估了从特定句法结构抽象的程度。这些指标共同为评估模型深度中语义概念层的形成提供了量化基础。

5.1.2 探测句法结构。除了语义探针外，还部署了一个句法探针来映射语法结构的逐层编码。该分析通过识别模型关注点从句法处理转向语义抽象的位置，来划定潜在概念层的边界。为此，我们采用已建立的NLP探测技术，使用监督分类器通过从隐藏状态预测语言属性来检测句法结构[14]。具体来说，我们使用线性分类器以确保探针主要提取表征中编码的信息，而不是依赖其学习任务的能力。先前的工作[13, 36]表明，线性探针的高分类准确率表明句法信息在LLM的表征空间中是线性可分的，证明了模型组织此类结构的能力。

执行以下步骤以量化句法信息在每个层被显式编码的程度：

(1) 数据准备：将HumanEval-X数据集中的每个代码样本解析为其对应的AST。
(2) 词元到节点对齐：通过为输入片段中的每个词元识别对应的AST节点来构建带标签的数据集。此步骤创建训练实例，其中输入特征是来自给定层的词元隐藏状态向量，目标标签是其关联的AST节点类型（例如，FunctionDef、ForStmt、BinOp）。
(3) 逐层探针训练：对于冻结模型的每个Transformer层，训练一个线性分类器（多项逻辑回归）。底层模型权重保持不变；仅更新探针的参数。探针旨在从该层的单个词元隐藏状态向量预测AST节点类型。

# 5.2 结果

我们同时应用基于RSA的语义探针和基于AST节点预测的句法探针，以衡量分层表征对词汇和句法变化的稳定性。我们将概念层定义为同时满足两个标准的层区域：(i) 它们在语内和跨语言转换下均表现出较高的语义不变性，以及 (ii) 它们具有较低的句法可解码性，这意味着细粒度的句法细节不再能够线性恢复。接下来我们总结这两个探针如何共同揭示此类层。

**语义抽象探针结果**。图2显示了词汇变化下的语内不变性，通过原始代码表征与变量重命名后代码表征之间的余弦相似度来衡量。在Llama-3.1-8B中，相似度在约第8至15层的中部层带最高，而在Qwen2.5-Coder-32B中，相似度在第31至43层持续保持高位。这些层带表明，对应层对每种语言内部的标识符变化具有鲁棒性。

图3展示了句法变化下的跨语言语义等价性，显示了同一算法在不同语言中实现时其表征间的平均余弦相似度。同样，两个模型都在中间层达到最高或接近最高的相似度：Llama-3.1-8B在第8至15层，Qwen2.5-Coder-32B则在第31至43层形成一个更宽的平稳区。虽然有些曲线峰值稍早出现或缓慢衰减，但这些范围对应的是跨语言语义相似度稳定较高的连续区域，而非单一的尖锐最大值。这表明这些层编码了与语言无关、算法级别的结构，抽象掉了表面的词汇和句法选择。

总体而言，语义探针表明，两个模型的中间层都形成了与语言无关的表征，这些表征捕获了核心的算法逻辑，同时很大程度上忽略了表面的变化。

**句法结构探针结果**。如图4所示，对于两个模型，显式句法信息的可解码性都遵循一个独特的U形曲线。AST节点预测的准确率在最早几层最高，表明较低层受句法主导强烈，执行浅层解析。随后性能急剧下降，在Llama-3.1-8B中约第9层、在Qwen2.5-Coder-32B中约第36层达到最低点。这种下降表明细粒度句法特征变得不那么线性可分，并在中间层以更抽象的方式编码。最后，准确率在更高层持续恢复，这与模型在代码生成过程中需要重建符合句法的输出相一致。

综合来看，图2至图4揭示了跨层语义抽象与句法保真度之间的反向关系。早期层结合了相对较高的语义相似度和较高的AST预测准确率，反映了强烈的表层句法编码。相比之下，中间层是首个高语义相似度与最低句法可解码性同时出现的区域，因此我们将这些层带指定为模型的概念层区域。最后，上层恢复了生成所需的句法信息，导致AST准确率再次上升，而语义相似度逐渐下降。

更具体地说，峰值（或接近峰值）语义相似度与最低句法可解码性的交汇界定了概念层。例如，在Llama-3.1-8B中，第9层的句法准确率最低点落在第8至15层观察到的语义相似度层带内。类似地，在Qwen2.5-Coder-32B中，第36层的句法最低点位于跨越第31至43层的语义平稳区内。因此，我们将这些区间视为近似的概念层区域，在此区域中，代码被转换为一种通用的、以逻辑为中心的表征，然后在后续层中转换回特定语言的句法结构。

**发现2：** 代码语言模型以分层方式组织知识，在其中间层形成独特的“概念层”。这些层具有两个互补的特性：它们对词汇和句法变化表现出峰值不变性，但对细粒度句法细节的保留最少。

# 6.1 面向代码生成的神经元引导微调方法

6.1.1 研究设计。像LoRA这样的参数高效微调（PEFT）方法虽然有效，但通常会修改大量参数而不考虑其功能角色。这可能导致次优性能和灾难性遗忘，即模型在微调后丧失先前学习任务的能力[23]。一种更具针对性、机制性的微调方法可以通过保留先验知识同时提升目标任务性能来解决这些问题。

在代码生成中，有效的适应需要学习目标编程语言的句法结构和惯例。基于我们对语言特定神经元的研究发现（RQ1），我们提出了一种神经元引导的微调方法，该方法仅更新包含特定神经元的模块中的参数。这种稀疏微调[58]通过保留模型的其余部分，在保持通用代码智能的同时，提升了目标任务上的性能。

该过程包含三个步骤：

(1) 语言特定神经元识别：使用PLS分数选择针对目标语言特化的神经元；
(2) 参数冻结：冻结除承载特定神经元的模块（例如，像gate_proj这样的线性层）之外的所有参数；

6.2.1 研究设计。语义（IV型）代码克隆的检测——即那些在实现、语法和词汇上存在显著差异但功能相同的代码片段——仍然是一个重大挑战[47]。依赖表层特征的现有方法通常无法捕捉这种深层的语义等价性[37]。概念层（在RQ2中被识别为编码算法逻辑的抽象且与语言无关的表示）为此任务提供了理想的表示基础。

# 6.3 面向代码摘要的概念层引导迁移学习

6.3.1 研究设计。为低资源编程语言的代码生成高质量的自然语言摘要具有挑战性，原因在于严重的数据稀缺性以及预训练知识的灾难性遗忘[23]。我们利用概念层（第5节）——即程序逻辑的与语言无关的表示——作为语义桥梁，以实现跨语言迁移。

为了高效地适配模型，我们使用LoRA[17]，并在所有概念层和更高层的自注意力模块中插入可训练的低秩适配器，同时冻结所有原始参数。这使参数更新集中在高层语义处理上，并保留了预训练期间获得的句法知识。

训练分两个阶段进行：

(1) 高资源适配：我们首先在大规模高资源语言（Python和Java）的代码摘要数据集上，通过文档字符串标记的交叉熵损失来优化LoRA适配器。此步骤教会模型将代码语义映射到自然语言描述。
(2) 低资源迁移：随后，我们在目标低资源语言（Ruby）的有限数据上继续训练相同的适配器，同样使用交叉熵损失。通过复用在高资源语言上学到的概念层表示，模型能够以远少于从头开始训练的数据量来适应新的语法。与传统的多语言微调[4, 49]相比，这种顺序方法提供了一种更高效、更直接的跨语言迁移形式。

我们将基于概念层的迁移学习方法与四种基线进行比较：(1) 未经任何任务特定调优的基础模型；(2) 直接法，该方法针对每种语言，在广泛的层上独立进行单阶段LoRA微调，仅使用该语言的数据；(3) 概念层LoRA，该方法仅将单阶段LoRA应用于已识别的概念层，并仅使用目标低资源语言（Ruby）的数据进行训练；(4) 两阶段LoRA，这是一个更强的基线，遵循与我们方法相同的高资源→低资源训练计划，但将LoRA适配器置于广泛的层上，而非限制在概念层。与之前的实验一样，Llama-3.1-8B作为基础模型，我们使用BLEU-4和ROUGE-L在CodeSearchNet[19]上进行评估。

6.3.2 结果。代码摘要结果报告于表5。对于我们指定的低资源语言Ruby，所提出的两阶段概念层迁移方法在所有方法中取得了最佳性能，BLEU-4为16.15，ROUGE-L为31.80。这相对于直接按语言LoRA（BLEU-4为14.80）有 $9 . 1 \%$ 的提升，相对于仅使用Ruby训练的单阶段概念层LoRA模型（BLEU-4为13.09）有 $2 3 . 4 \%$ 的相对提升。它也超越了更强的两阶段LoRA基线（BLEU-4为14.66，ROUGE-L为30.18），这表明将适配集中在概念层上，能够带来超越高资源-低资源训练计划本身所提供的额外收益。

# 7 局限性与有效性威胁



本研究存在若干可能影响研究结果有效性的局限性：

神经元的多元性。本研究通过分析多语言代码输入下神经元的激活情况，探究了代码大语言模型中编程语言特定神经元的分布。然而，先前的研究[7]表明，单个神经元可能具有多元性，可能编码多个不同或不相关的概念。尽管本文识别的神经元已通过因果验证，但由于某些神经元可能参与重叠的功能，对其解释可能仍不完整且不精确。对此类多元神经元的深入考察将留待未来工作。

模型与语言的普适性。我们的研究结果基于两种特定的密集Transformer架构和五种编程语言。这些结论在多大程度上能推广到显著不同的架构（例如，混合专家模型、参数量超过700亿的模型）、专有的闭源模型或其他编程范式，尚需进一步研究。

缺乏基准事实。由于代码模型可解释性领域缺乏标准的“基准事实”，难以客观评估解释的质量[60]。为解决此问题，我们从两个角度验证了结果：(1) 停用已识别的语言特定神经元，并测量其对代码生成的影响；(2) 展示语言特定神经元和概念层如何为多语言代码智能的训练策略和代码呈现方法提供信息。

# 8 相关工作

大多数针对代码大语言模型的可解释性研究都集中在注意力机制的分析上[40, 42, 59]。这些研究通过检查注意力权重和激活值，来理解模型在整个输入序列中的关注点。Mohammadkhani 等人[30]分析了 CodeBERT 和 GraphCodeBERT 的注意力分数，揭示了这些模型在软件工程任务中如何将注意力分配到代码的不同部分。Liu 等人[26]利用注意力可解释性研究了 CodeT5 和 CodeGPT 等模型在生成、翻译和修复过程中如何关注代码元素。Paltenghi 等人[34]比较了神经代码模型与人类开发者的注意力模式。Wan 等人[52]结合注意力分析、词嵌入探测和语法树归纳来理解代码结构和语义。

另一研究方向侧重于表征和概念分析，旨在探究代码大语言模型如何在内部组织和表示编程知识。Sharma 等人[41]揭示了编码概念如何在神经元间冗余分布。Ma 等人[28]使用费舍尔信息展示了多语言代码模型如何根据结构相似性对语言进行分组。Haider 等人[12]在前馈网络中识别了从语法到语义的层次化概念编码。Kargaran 等人[22]对 Llama 模型使用逻辑透镜和神经元激活分析，研究了多种编程语言与英语之间的共享表征。他们发现模型的内部概念空间更接近英语，并且虽然语言特定的神经元集中在底层，但专属于单一编程语言的神经元往往出现在顶层。Wang 等人[55]提取了驱动模型预测的最小关键代码片段。Troshin 等人[50]定量评估了程序属性在不同层中的编码方式。

最后，因果与反事实分析作为一种新兴方法，旨在通过针对性干预来识别代码大语言模型行为中的因果关系。Nader-Palacio 等人[33]为代码模型中的因果推理建立了形式化基础。Gupta 等人[10]将此扩展到多模态代码生成，建模了不同输入如何因果性地影响输出。Cito 等人[2]通过生成最小的代码扰动来揭示模型的决策边界。Hooda 等人[15]通过改变程序逻辑并观察模型响应来测试概念理解。Rodriguez-Cardenas 等人[38]评估并比较了用于代码模型解释的因果方法。

我们的工作坚定地属于表征与概念分析范式。与先前研究相比，我们首次系统分析了模型对多语言代码输入的神经元激活模式，并评估了各个层在整个输出生成过程中的贡献。此外，我们研究了已识别的编程语言特定神经元的因果效应，并利用这些洞见来策略性地控制模型行为，从而提升代码相关任务的性能。

# 9 结论



本文中，我们提出了针对代码大语言模型的神经元级可解释性研究，揭示了编程语言特定神经元与跨语言概念层的独特作用。我们的发现为多语言代码理解与生成的内在机制提供了更深入的见解。我们进一步通过神经元引导微调、概念感知嵌入和迁移学习，展示了这些见解的实际应用价值——在包括代码生成、克隆检测和摘要在内的多项任务中持续提升性能。通过增强代码语言模型的透明度和可控性，本研究为软件工程领域可解释人工智能的未来研究奠定了基础，并支持多语言代码智能的进一步发展。
